上手跑的出的几个主要问题
==========================================

1. 代码和文档里有残留的个人路径和内部地址没清。JobShop README 写死了 /data_storage/chihh2311/ 的 conda 路径（5 处），v1_task_run_guide_zh-CN.md 原样抄了；python.py:51 conda fallback 里有 /mnt/shared-storage-user/p1-shared/luotianwei/ 的个人目录；batch config 和 pr_review.py 的 API 默认值是内部的 litellm.nbdevenv.xiaoaojianghu.fun，外部用户访问不了还会静默 fallback。

2. 环境体系没有统一文档。实际要用 7 个 conda 环境，但主 README 只提了 init.sh 建 frontier-eval-2；哪个 benchmark 对应哪个 env、use_conda_run 和 python_path 两种模式什么时候该用哪种，都没有集中说明，信息散在 env_specs、batch yaml 和各任务目录里。PYTHONNOUSERSITE=1 这个对环境隔离很关键的变量，脚本里都在用但 README 没提。

3. frontier-v1-main 漏了 Optics 的 requirements.txt 导致 16 个任务全挂；MolecularMechanics 实测还需要 openff-toolkit 等包但不在任何 requirements.txt 里，只有翻 env_specs 才知道要用 openff-dev 环境。

4. third_party/ShinkaEvolve/ 是空目录但被 git track 了，新人会以为依赖已就位。patches/ 下有对应 patch 但 quickstart 没把 clone + patch 列为必要步骤。docs/v1_task_run_guide_zh-CN.md 也已 commit 但里面全是特定机器路径，该清理或删除。

5. 外部资产和 task-local 依赖没有收敛成 checklist。dc-rl 要手动 clone + patch，PhySense 要数据和 checkpoint，DuckDB 和 EV2Gym 要额外装 verification 依赖，ReactionOptimisation 的 summit 环境 pip 直接 resolution-too-deep——这些分散在各自任务目录里，没有一个入口文档告诉新人"跑全量之前还差这些步骤"。
