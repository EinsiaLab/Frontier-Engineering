# 任务：带容量约束的车辆路径 — A-n32-k5

## 问题描述

在 `baseline/init.py` 中实现 `solve(instance)` 函数。

给定一个配送中心（索引 0）和 31 个客户（各有需求量），将客户分配到若干条路线，使得：

1. 每个客户**恰好出现在一条路线**中。
2. 每条路线的需求量之和不超过**容量 = 100**。
3. 所有路线的总 EUC_2D 距离（配送中心 → 客户 → 配送中心，累加所有路线）**最小**。

## 接口

```python
def solve(instance: dict) -> list[list[int]]:
    ...
```

| 键         | 类型               | 说明                         |
|------------|------------------|------------------------------|
| `coords`   | `list[(int,int)]` | (x, y) 坐标；索引 0 = 配送中心 |
| `demands`  | `list[int]`       | 每节点需求量；demands[0] = 0   |
| `capacity` | `int`             | 车辆容量 = 100                |

返回路线列表，每条路线为客户索引列表（1…31），不含配送中心索引 0。

## 评测

距离使用 EUC_2D：`round(sqrt((x1-x2)²+(y1-y2)²))`。

```
score = min(1.0, 784.0 / total_distance)
```

`valid = 1` 要求：索引合法、无重复访问、无容量超限、覆盖全部 31 个客户。

## 人类最优

A-n32-k5 已知最优距离：**784**（score = 1.0）。
来源：Augerat et al. (1995)，http://vrp.atd-lab.inf.puc-rio.br/
